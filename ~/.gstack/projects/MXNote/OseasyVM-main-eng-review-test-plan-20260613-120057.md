## Affected Pages/Routes

- **pages/Index** (编辑器主页) — TextArea编辑区 + Markdown预览切换 + 同步状态指示器
- **会话发现弹窗** — "检测到附近设备上的活跃笔记，是否继续编辑？"

## Key Interactions to Verify

- 单设备编辑：输入文本 → Markdown预览模式切换 → 格式正确渲染
- 接力编辑核心流：设备A输入文本 → 设备B打开应用 → 发现活跃Session → 确认接管 → 光标位置正确 → 继续输入
- 光标同步：设备A移动光标 → 150ms后设备B看到光标位置更新 → 设备B点击文本区域 → 接管编辑焦点
- 分块同步：输入>500KB长文本 → 自动分块 → 设备B收到所有块后组装 → 内容完整一致
- 离线队列：设备A离线编辑 → 设备A恢复网络 → 自动同步到设备B

## Edge Cases

- **分布式总线不可用**: 应自动降级到纯本地模式，显示离线指示器
- **500KB边界**: 刚好500KB的笔记不分块直接同步，500001字节自动切分
- **快速切换设备**: 设备A→设备B→设备A连续三次接力，每次会话状态正确恢复
- **同步中断**: 同步过程中关闭应用 → 重新打开 → 不丢失不重复
- **空笔记**: 新建空白笔记 → 接力到另一设备 → 空白状态正确
- **特殊字符**: Markdown源码含中文、Unicode、emoji → 同步后编码一致
- **并发编辑冲突**: 设备A和设备B同时编辑 → LWW裁决 → 冲突副本保留
- **30s时钟偏差窗口**: 两次编辑时间差在30s内 → 两个版本都保留 → 用户手动裁决

## Critical Paths

- 设备A打开笔记输入 → 设备B靠近自动检测 → 确认接力 → 光标同步 → 继续输入 → 两端内容一致
- 设备A编辑过程 → 分布式总线断开 → 降级本地 → 恢复连接 → 自动合并离线修改
